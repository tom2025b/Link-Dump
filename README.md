## Resource metadata

Every resource has six fields:

| Field | What it tells you |
|---|---|
| Type | Whether the source is official documentation, an article, blog, book, video, or reference. |
| Title | The current title verified against the destination. |
| URL | A direct HTTPS destination; generic search-result pages are not accepted. |
| Why this is valuable | The resource's distinct teaching role, when to use it, and important limits or safety context. |
| Difficulty | Beginner, Intermediate, or Advanced. |
| Estimated time | A realistic first-pass estimate or `Ongoing reference`. |

The collection is **official-first, not official-only**. Maintainer guides,
specifications, and canonical books are preferred; community material remains
when it provides a teaching role that the primary reference does not.

## Maintenance standard

When adding or reviewing a resource:

1. Prefer the current primary source.
2. Confirm the title, direct URL, scope, and authority—not just the status code.
3. Add a distinct teaching role; do not duplicate an existing normalized URL.
4. Include all six metadata fields and place the resource in its strongest
   primary section.
5. Record a concise concept note and future gap when no durable resource clears
   the quality bar.
6. Recheck operational advice against a threat model, rollback path, least
   privilege, and restore or recovery evidence.
